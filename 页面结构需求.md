# 页面结构需求

本文件用于迁移用户自定义页面内容。只描述 `blog/pages/` 下的页面目录、Markdown 属性区和图片放置规则。`blog/example/pages/` 只放示例页面，用户不要直接在 example 目录里维护正式内容。

## 目录结构

每个自定义页面一个独立目录，目录名就是页面 URL slug。当前只支持一级页面路径。

```text
blog/pages/
└─ {slug}/
   ├─ index.md
   └─ img/              # 可选，存放当前页面正文图片
      ├─ {slug}-1.webp
      └─ {slug}-2.webp
```

示例：

```text
blog/pages/
└─ demo-page/
   ├─ index.md          # 生成 /demo-page
   └─ img/
      └─ demo-page-1.svg
```

## 文件命名

| 文件 | 要求 |
| --- | --- |
| `blog/pages/{slug}/index.md` | 页面正文和属性区。 |
| `blog/pages/{slug}/img/` | 可选页面图片目录。 |
| `img/{slug}-1.webp` | 建议的正文第 1 张图片命名。 |
| `img/{slug}-2.webp` | 建议的正文第 2 张图片命名，后续按顺序递增。 |

页面目录名建议只使用小写字母、数字和短横线，例如 `about`、`projects`、`contact-me`。

## 属性区

Markdown 文件开头两条 `---` 之间是属性区。页面属性区只需要站内页面本身的信息，不写文章字段。

```md
---
title: "示例页面"
description: "一个用于验证自定义页面结构和图片引用的示例页面。"
comments: false
---

这里写页面正文。页面正文适合放关于、项目、联系方式等固定内容。

![页面图片说明](./img/demo-page-1.svg)
```

| 字段 | 要求 |
| --- | --- |
| `title` | 页面标题，必填。 |
| `description` | 页面摘要，用于 SEO。 |
| `comments` | 是否开启评论，默认 `false`。当前只保留字段，不开发评论 UI。 |

页面不要写 `slug`、`date`、`categories`、`tags`、`cover`、`top`。这些字段只属于文章。

## 正文图片引用

页面专用图片放在当前页面目录的 `img/` 内，用相对路径引用。

```md
![图片说明](./img/demo-page-1.svg)

![图片说明](./img/demo-page-2.webp)
```

如果图片会被多个页面或文章复用，可以放在 `blog/public/`。该目录构建后使用根路径访问，例如 `blog/public/assets/banner.webp` 在 Markdown 中写作 `/assets/banner.webp`。
